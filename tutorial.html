NextJS
Bite-Sized Guide

Quick guide and introduction to NextJS.

About

The software resource that I have elected to create a tutorial for is NextJS by Vercel. Next.js is a framework of React, which is a Javascript library for building user interfaces. Some notable websites, services, or applications built using the framework are Hulu, Staples, Nike, the mobile version of Twitch, among other popular sites as well.

Why I feel that this software resource may have relevance to my topic is because I feel that I am most likely to pursue a topic that would be brought to fruition as an online application, as in, it is easily accessible by all versus something that would require installs.

I particularly am partial to pursuing further information on this software resource because I have previously and am currently working with it in a personal project. The pursuit of continuing to use it is purely based in the fact that I would like to hone my skills with it and use it as effectively as possible, like how other sites and applications have done.

Features

I feel that the visual features and ability integrated into Next.js are great for a good variety of topics that I may want to pursue. I am fairly certain that I would like to pursue an online application that is easily accessible to all and using this, that becomes fairly easily achievable. Particularly in the case of data analysis, there is easy protocol for data fetching that make things much simpler and easier in regards to achievement.

Next.js, as previously mentioned, is a framework of React. Next.js, however, has the ability to build server-side rendered applications. Next.js functions similarly to other frameworks, such as Angular. They surely have their similarities, but when it came down to preference the features and overall functionality won me over.

Obtaining the Resource

I found this resource when looking for a framework that had the ability to build server-side rendered applications. It is open source, but not explicitly stated on their website or documentation, however clarified with their MIT license. Installation is rather simple, a guide is provided here but in quick explanation it is:

npx create-next-app@latest

This generates a blank project with everything already set up. If you wish to have this project be Typescript based, you would us the flag --typescript.

Alternatively, if you would like to set it up manually, you would follow the manual setup on the Getting Started guide.

Setup

Truthfully, you could the software to run fairly easily out of the box when using the standard setup command for Next.js, which is npx create-next-app@latest as previously mentioned.

The only further step that has to be taken for deployment is actual deployment (like, through a service) or test deployment enabled through:

npm run dev

Next.js is a framework of React, heavily relying on Javascript (or .jsx, .ts, .tsx, et cetera), which explains the heavy involvement of using npm and npx for package and dependency handling for it.

Other than that, setup relies on your own manual configurations in creating the application.

Execution

Getting the resource ready to use is fairly simple. With the automatic setup option, Next.js runs straight out of box. For development purposes, the demonstration of the application can be done with npm run dev and final deployment could be handled by however you would elect to host your application.
